232: /*E
233:     SNESConvergedReason - reason a `SNESSolve()` was determined to have converged or diverged

235:    Values:
236: +  `SNES_CONVERGED_FNORM_ABS` - 2-norm(F) <= abstol
237: .  `SNES_CONVERGED_FNORM_RELATIVE` - 2-norm(F) <= rtol*2-norm(F(x_0)) where x_0 is the initial guess
238: .  `SNES_CONVERGED_SNORM_RELATIVE` - The 2-norm of the last step <= stol * 2-norm(x) where x is the current
239: .  `SNES_DIVERGED_FUNCTION_COUNT` - The user provided function has been called more times than the maximum set in `SNESSetTolerances()`
240: .  `SNES_DIVERGED_DTOL` - The norm of the function has increased by a factor of divtol set with `SNESSetDivergenceTolerance()`
241: .  `SNES_DIVERGED_FNORM_NAN` - the 2-norm of the current function evaluation is not-a-number (NaN), this
242:       is usually caused by a division of 0 by 0.
243: .  `SNES_DIVERGED_MAX_IT` - `SNESSolve()` has reached the maximum number of iterations requested
244: .  `SNES_DIVERGED_LINE_SEARCH` - The line search has failed. This only occurs for `SNES` solvers that use a line search
245: .  `SNES_DIVERGED_LOCAL_MIN` - the algorithm seems to have stagnated at a local minimum that is not zero.
246: -  `SNES_CONERGED_ITERATING` - this only occurs if `SNESGetConvergedReason()` is called during the `SNESSolve()`

248:    Level: beginner

250:     Notes:
251:    The two most common reasons for divergence are an incorrectly coded or computed Jacobian or failure or lack of convergence in the linear system (in this case we recommend
252:    testing with `-pc_type lu` to eliminate the linear solver as the cause of the problem).

254:    `SNES_DIVERGED_LOCAL_MIN` can only occur when using the line-search variant of `SNES`.
255:        The line search wants to minimize Q(alpha) = 1/2 || F(x + alpha s) ||^2_2  this occurs
256:        at Q'(alpha) = s^T F'(x+alpha s)^T F(x+alpha s) = 0. If s is the Newton direction - F'(x)^(-1)F(x) then
257:        you get Q'(alpha) = -F(x)^T F'(x)^(-1)^T F'(x+alpha s)F(x+alpha s); when alpha = 0
258:        Q'(0) = - ||F(x)||^2_2 which is always NEGATIVE if F'(x) is invertible. This means the Newton
259:        direction is a descent direction and the line search should succeed if alpha is small enough.

261:        If F'(x) is NOT invertible AND F'(x)^T F(x) = 0 then Q'(0) = 0 and the Newton direction
262:        is NOT a descent direction so the line search will fail. All one can do at this point
263:        is change the initial guess and try again.

265:        An alternative explanation: Newton's method can be regarded as replacing the function with
266:        its linear approximation and minimizing the 2-norm of that. That is F(x+s) approx F(x) + F'(x)s
267:        so we minimize || F(x) + F'(x) s ||^2_2; do this using Least Squares. If F'(x) is invertible then
268:        s = - F'(x)^(-1)F(x) otherwise F'(x)^T F'(x) s = -F'(x)^T F(x). If F'(x)^T F(x) is NOT zero then there
269:        exists a nontrivial (that is F'(x)s != 0) solution to the equation and this direction is
270:        s = - [F'(x)^T F'(x)]^(-1) F'(x)^T F(x) so Q'(0) = - F(x)^T F'(x) [F'(x)^T F'(x)]^(-T) F'(x)^T F(x)
271:        = - (F'(x)^T F(x)) [F'(x)^T F'(x)]^(-T) (F'(x)^T F(x)). Since we are assuming (F'(x)^T F(x)) != 0
272:        and F'(x)^T F'(x) has no negative eigenvalues Q'(0) < 0 so s is a descent direction and the line
273:        search should succeed for small enough alpha.

275:        Note that this RARELY happens in practice. Far more likely the linear system is not being solved
276:        (well enough?) or the Jacobian is wrong.

278:    `SNES_DIVERGED_MAX_IT` means that the solver reached the maximum number of iterations without satisfying any
279:    convergence criteria. `SNES_CONVERGED_ITS` means that `SNESConvergedSkip()` was chosen as the convergence test;
280:    thus the usual convergence criteria have not been checked and may or may not be satisfied.

282: .seealso: [](chapter_snes), `SNES`, `SNESSolve()`, `SNESGetConvergedReason()`, `KSPConvergedReason`, `SNESSetConvergenceTest()`, `SNESSetTolerances()`
283: E*/
